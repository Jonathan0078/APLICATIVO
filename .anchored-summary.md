## Key Decisions
- `i18n.t()` only does key lookup — no parameter interpolation. Use separate keys per unique message.
- Store i18n key names in data objects (e.g., `materialsDb.desc = "retentor.mat_general"`) and translate at display time with `i18n.t(selectedMat.desc)`
- Seal type descriptions/usage moved to dynamic `i18n.t('retentor.desc_' + sealKey)` pattern
- For GreasePRO: individual i18n keys per alert message (e.g., `grease.gq.err_diameter`)
- GreasePRO HTML files load i18n.js at bottom of `<body>` — available when user triggers alerts
